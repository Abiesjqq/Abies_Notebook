替换策略：

- FIFO：先进先出
- LRU：替换距离上次使用时间最长的
- OPT：替换未来访问最晚的

循环访问的情况下，FIFO和LRU可能全miss

堆栈式替换算法：用 $B_t(n)$ 表示时间t下cache中存储的block，其中n为每个set的block数。若满足 $B_t(n)\in B_{t+1}(n)$，则称为堆栈式替换算法。

!!! examples ""

    访问序列：2 3 2 1 5 2 4 5 3 2

    |       | 2   | 3   | 2   | 1   | 5   | 2   | 4   | 5   | 3   | 2   |
    | ----- | --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
    | St(1) | 2   | 3   | 2   | 1   | 5   | 2   | 4   | 5   | 3   | 2   |
    | St(2) |     | 2   | 3\* | 2   | 1   | 5   | 2   | 4   | 5   | 3   |
    | St(3) |     |     |     | 3   | 2   | 1\* | 5   | 2\* | 4   | 5   |
    | St(4) |     |     |     |     | 3   | 3   | 1   | 1   | 2   | 4\* |
    | St(5) |     |     |     |     |     |     | 3   | 3   | 1\* | 1   |
    | St(6) |     |     |     |     |     |     |     |     |     |     |

Belady现象：随着n增大，hit数反而减小（FIFO）

堆栈的硬件实现：用1 bit表示两两访问的先后顺序，n(n-1)/2 个触发器能表示n个block的顺序。

write hit策略：

1. write through：cache和内存中始终一致
2. write back：只写cache，并维护dirty bit，必要时（被替换时）才写回内存

write miss策略：

1. no write allocate：和write through对应，只写memory。下一步read时从memory读。
2. write allocate：和write back对应，先在memory中调入cache，再写cache

dirty bit的处理：

如果cache中写入，但未写回memory，dirty位置一。

!!! example ""

    操作序列：

    ```
    write Mem[100]
    write Mem[100]
    read Mem[200]
    write Mem[200]
    write Mem[100]
    ```

    write allocate + write back:

    |      | cache      | memory |
    | ---- | ---------- | ------ |
    | miss | y(d)       | n      |
    | hit  | y(d)       | n      |
    | miss | write back |        |
    | hit  | y(d)       | n      |
    | miss | y(d)       | n      |

    write allocate + write through:

    |      | cache | memory |
    | ---- | ----- | ------ |
    | miss | n     | y      |
    | miss | n     | y      |
    | miss |       |        |
    | hit  | y     | y      |
    | miss | n     | y      |
